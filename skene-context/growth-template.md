## Lifecycle Stages

### 1. DISCOVERY: Initial Awareness

#### Milestones

1. **Website Visit**: User visits the Skene Growth website or documentation.
2. **Github Stargazing**: User stars the Skene Growth repository on Github
3. **Documentation Read**: User reads a significant part of the documentation to understand the CLI's capabilities.

#### Key Metrics

- **Website Conversion Rate**: Number of users visiting the site who become users / Total website visitors (Benchmark: > 5%)
- **Github Stars**: Count of total stars on the project repository (Benchmark: > 100 stars/month)
- **Documentation Engagement**: Number of users visiting documentation / Total website visitors (Benchmark: > 20%)

---

### 2. ACTIVATION: First Use

#### Milestones

1. **Installation**: User successfully installs the Skene Growth CLI tool.
2. **First Command Execution**: User runs their first command (e.g., `skene-growth plan`).
3. **Successful Template Generation**: User creates a basic growth template using the CLI.
4. **Growth Loop Plan Generated**: User runs the plan command

#### Key Metrics

- **Installation Success Rate**: Number of successful installations / Total download attempts (Benchmark: > 90%)
- **Time to First Command**: Time elapsed between installation and first command execution. (Benchmark: < 1 hour)
- **Template Generation Rate**: Number of users generating a template / Total number of users running the CLI. (Benchmark: > 30%)
- **Successful Plan Generation**: Number of users who successfully generate a plan / Number of installations (Benchmark: > 50%)

---

### 3. ADOPTION: Regular Usage

#### Milestones

1. **Daily Log Integration**: User integrates daily logs into their workflow.
2. **Growth Objective Definition**: User defines growth objectives using the CLI.
3. **Weekly Active Users**: User uses the CLI at least once per week

#### Key Metrics

- **Daily Log Integration Rate**: Number of users integrated / Total number of users (Benchmark: > 20%)
- **Objective Definition Rate**: Number of users defining objectives / Total number of users (Benchmark: > 40%)
- **Weekly Active Users (WAU)**: Number of unique users using the CLI at least once a week. (Benchmark: > 50% of installed base)
- **Command Usage**: Average number of commands executed per user per week. (Benchmark: > 5 commands/week)

---

### 4. RETENTION: Long-term Value

#### Milestones

1. **Continued Usage**: User consistently uses the CLI over a period of several months.
2. **Community Engagement**: User participates in the community forum or contributes to the project.
3. **Feedback Submission**: User provides feedback or feature requests for the CLI.

#### Key Metrics

- **Monthly Active Users (MAU)**: Number of unique users using the CLI at least once a month. (Benchmark: > 30% of installed base)
- **User Retention Rate**: Percentage of users who continue using the CLI after 3 months. (Benchmark: > 60%)
- **Community Contribution Rate**: Number of users contributing to the community / Total number of users. (Benchmark: > 5%)
- **Active Community**: Number of forum posts and issues created/commented on (Benchmark: > 5 posts and comments a week)
