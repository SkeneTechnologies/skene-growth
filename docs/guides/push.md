# Push Command

The `push` command pushes pre-generated engine, feature registry, and trigger artifacts to Skene Cloud upstream.

`push` no longer builds migrations. Generate artifacts first with `build`.

## Prerequisites

Before running `push`, you need:

- `skene/engine.yaml` generated or updated by `build`
- `{output_dir}/feature-registry.json` (default `skene-context/feature-registry.json`) from `build` or analyze â€” if missing, push continues but omits the registry from the package until you run `build`
- `supabase/migrations/*_skene_triggers.sql` generated by `build` (older repos may still use `*skene_trigger*` or `*skene_telemetry*` filenames; push picks the latest eligible migration)

`output_dir` comes from `.skene.config` (see [configuration](configuration.md)).
- Upstream authentication via `skene login` (or `SKENE_UPSTREAM_API_KEY`)

## Basic usage

Push existing artifacts:

```bash
uvx skene push
```

Push to upstream (Skene Cloud):

```bash
uvx skene push --upstream https://skene.ai/workspace/my-app
```

`--push-only` is now a legacy no-op because push is always artifact-only.

## How it works

### Step 1: Validate local artifacts

Push checks that:

- `skene/engine.yaml` exists
- A trigger migration exists in `supabase/migrations/` (typically `*_skene_triggers.sql` from `build`)

If missing, push exits with guidance to run `skene build`.

### Step 2: Build package payload

Push packages:

- engine YAML content
- feature registry JSON (same path `build` uses: `{output_dir}/feature-registry.json`)
- latest trigger migration SQL content

### Step 3: Push upstream

When an upstream URL and token are available, push sends the package to upstream deploy API.

## Artifact source

Trigger SQL is generated by `build` from `engine.yaml` features that include `action`.
Features without `action` are treated as code-only and do not produce trigger SQL.

## Upstream authentication

To push to upstream, authenticate first:

```bash
uvx skene login --upstream https://skene.ai/workspace/my-app
```

The upstream URL can also be resolved from:

1. `upstream` field in `.skene.config` (saved by `skene login`)
2. `--upstream` CLI flag (highest priority)
